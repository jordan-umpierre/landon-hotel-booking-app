# Landon Hotel Booking App

![Landon Hotel Booking App banner](src/main/UI/src/assets/images/hotel/splash_hotelphoto.jpg)

A full-stack hotel reservation demo built with Spring Boot and Angular. Users can browse the hotel UI, search available rooms by check-in and checkout date, create reservations, view localized welcome messages in English and French, and see the same event time rendered across Eastern, Mountain, and UTC time zones.

## What It Demonstrates

- Spring Boot REST API design for room availability and reservation workflows
- Angular 14 frontend integration served by the Spring Boot application
- Spring Data JPA persistence with an H2 database for local development
- DTO/converter-based request and response mapping for reservation data
- Concurrent localization loading with Java threads and resource bundles
- Time zone conversion with Java time APIs
- Dockerized packaging for a one-command container run

## Highlights

- Search rooms by check-in and checkout dates
- Create reservations through a JSON API
- Display localized English and French welcome messages
- Display a shared event time in ET, MT, and UTC
- Inspect local data with the H2 console
- Build and run the app locally or in Docker

## Run With Docker

From the repository root:

```bash
docker build -t landon-hotel-booking-app .
docker run -p 8080:8080 landon-hotel-booking-app
```

Then open:

```text
http://localhost:8080
```

## Run Locally

Prerequisites:

- Java 17
- Node.js 18+
- npm

Install the Angular dependencies first:

```bash
cd src/main/UI
npm ci
cd ../../..
```

Start the Spring Boot app from the repository root:

```bash
./mvnw spring-boot:run
```

On Windows PowerShell:

```powershell
.\mvnw.cmd spring-boot:run
```

Then open:

```text
http://localhost:8080
```

The local H2 console is available at:

```text
http://localhost:8080/h2-console
```

Use JDBC URL `jdbc:h2:file:~/landon-hotel-booking-db`, username `sa`, and a blank password.

## Test And Verify

Run backend tests from the repository root:

```bash
./mvnw -q "-Dexec.skip=true" test
```

On Windows PowerShell:

```powershell
.\mvnw.cmd -q "-Dexec.skip=true" test
```

Run the frontend build and headless tests from the Angular app folder:

```bash
cd src/main/UI
npm ci
npm run build
npm run test:ci
```

The CI test script auto-detects a local Chromium-based browser. It works with Google Chrome or Microsoft Edge, and you can also set `CHROME_BIN` manually if your browser is installed in a custom location.

Run a full package build from the repository root after installing frontend dependencies:

```bash
./mvnw package
```

The Angular build writes browser assets into `src/main/resources/static`, which is ignored because it is regenerated by the build.

Manual smoke test checklist:

- Open `http://localhost:8080` and confirm the hotel page loads.
- Confirm English and French welcome messages render.
- Confirm ET, MT, and UTC event times render.
- Search for rooms with check-in and checkout dates.
- Reserve a room and confirm the request succeeds.
- Open the H2 console and confirm reservation data is persisted locally.

## API Overview

- `GET /api/messages` returns English and French welcome messages plus the event time in ET, MT, and UTC.
- `GET /room/reservation/v1?checkin=YYYY-MM-DD&checkout=YYYY-MM-DD` returns available rooms for the requested date range.
- `GET /room/reservation/v1/{roomId}` returns one room by ID.
- `POST /room/reservation/v1` creates a reservation for a selected room.
